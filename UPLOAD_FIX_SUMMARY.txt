REPEATED DATA UPLOAD FIX - COMPLETE ✅
================================================================================

PROBLEM:
  When uploading data repeatedly for the same product/period:
  - Scores never changed
  - Rankings stayed the same
  - Alerts/recommendations not updated
  - Predictions not recalculated

ROOT CAUSE:
  Old data was NOT deleted before processing new upload
  - Old raw_data stayed in database
  - Old features stayed
  - Old scores stayed
  - Dashboard showed old values

FIX IMPLEMENTED:
  
  1. ✅ DELETE old raw_data before inserting new
     File: backend/app/services/data_service.py
     Function: ingest_dataframe()
     
  2. ✅ DELETE old features/scores before recalculating
     File: backend/app/services/feature_engineering.py
     Function: reprocess_all()
     
  3. ✅ DELETE old predictions/alerts/recs before generating new
     File: backend/app/api/v1/data.py
     Function: _run_scoring_pipeline()
